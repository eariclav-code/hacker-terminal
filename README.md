# Hacker Terminal

Текстовая хакерская игра-квест на C# / .NET, стилизованная под терминал.
Играешь за анонимного хакера с позывным **GHOST**, который взламывает сеть
корпорации **Nortech Industries**, чтобы добыть доказательства сокрытия
токсичных выбросов на заводе в Северном секторе.

## Запуск

Требуется [.NET 10 SDK](https://dotnet.microsoft.com/).

```bash
cd HackerTerminal
dotnet run
```

Прогресс автоматически сохраняется в `save.json` после ключевых действий
и при выходе (`exit`). При следующем запуске он подхватывается сам.

## Команды

| Команда                  | Описание                                    |
|---------------------------|----------------------------------------------|
| `help`                     | список всех команд                           |
| `ls` / `dir`               | содержимое текущей папки                     |
| `cd <папка>`               | перейти в папку (`cd ..` — на уровень выше)  |
| `cat <файл>`               | прочитать файл                               |
| `decrypt <файл> <ключ>`    | расшифровать файл шифром Цезаря              |
| `hack <цель>`              | взломать цель (мини-игра Mastermind)         |
| `scan`                     | просканировать узел, найти новые подключения |
| `connect <узел>`           | подключиться к другому узлу сети             |
| `status`                   | текущий прогресс: уровень, очки, локация     |
| `clear` / `cls`            | очистить экран                               |
| `exit`                     | сохранить прогресс и выйти                   |

## Сетевая карта

Игра — это не одна файловая система, а несколько **узлов сети**
(машин), между которыми можно перемещаться командой `connect`. Каждый
узел определяет собственную виртуальную ФС, цель для взлома и (для
финального узла) список возможных концовок — всё это описано в
JSON-файлах в папке [`/quests`](quests), а не зашито в код.

Типичный цикл на одном узле:

1. `ls` / `cat` — читаешь файлы, ищешь пароли и подсказки;
2. `decrypt` — расшифровываешь зашифрованные файлы, если такие есть;
3. `hack <цель>` — взламываешь защищённую цель мини-игрой Mastermind;
4. `scan` — после успешного взлома открывает следующие узлы сети,
   если они существуют;
5. `connect <узел>` — переходишь на новый узел и повторяешь цикл.

## Взлом: мини-игра Mastermind

Команда `hack <цель>` запускает мини-игру в духе **Mastermind**:
терминал загадывает секретный код из 4 цифр (от 1 до 6), а ты пытаешься
его подобрать. После каждой попытки даётся обратная связь:

- `x` — цифра и позиция угаданы точно;
- `o` — цифра есть в коде, но стоит не на своём месте.

Взлом ограничен **и** числом попыток, **и** реальным таймером слежки —
если засечь тебя успеют раньше, чем код будет подобран, взлом
считается проваленным, даже если оставались свободные попытки. На
финальном узле неудачный взлом сразу завершает игру плохой концовкой.

## Концовки

На финальном узле (архив Nortech) доступны **4 разные концовки** — в
зависимости от того, как быстро и чисто прошёл финальный взлом, и от
решения, принятого по ходу:

- **Чистая утечка** — код подобран быстро, с большим запасом по
  попыткам и времени;
- **Частичная утечка** — код подобран, но впритык, кто-то ещё был на
  сервере одновременно с тобой;
- **Засечён** — таймер слежки истёк раньше, чем код подобран, игра
  заканчивается провалом;
- **Сделка** — если по пути найдено анонимное сообщение с
  предложением взятки, перед взломом можно принять деньги и уйти,
  не трогая архив.

## Прохождение (спойлеры)

Полное сюжетное обоснование — в [`docs/STORY.md`](docs/STORY.md).

1. **Узел `home-pc`, уровень 0.** Прочитай `hint.txt` и `home/notes.txt`,
   затем расшифруй зашифрованный файл:
   ```
   cd home
   decrypt secret.txt 3
   ```
   Внутри — ключ доступа `shadow42`.

2. **Уровень 1.** Взломай mainframe найденным ключом:
   ```
   hack mainframe
   ```
   Подбери 4-значный код в мини-игре Mastermind. При успехе откроется
   скрытая директория `/system/secret` и появится ключ `admin1234`.

3. **Уровень 2.** Прочитай `password.txt` и `briefing.txt` в
   `/system/secret`, затем просканируй узел и подключись к архивному
   серверу:
   ```
   cd system/secret
   cat briefing.txt
   scan
   connect nortech-core
   ```

4. **Финал.** На узле `nortech-core` прочитай `archive_index.txt` и
   `bribe.txt`, затем взломай хранилище:
   ```
   hack core-vault
   ```
   Если решишь пойти на сделку — согласись, когда терминал спросит.
   Если нет — постарайся уложиться в лимит попыток и таймер слежки,
   чтобы получить чистую концовку.

Команда `scan` в любой момент подсказывает, что делать дальше, если
застрял, а `status` показывает общий прогресс.

## Формат квестов (`/quests`)

Весь сюжет, файлы, пароли, коды взлома и тексты концовок вынесены из
кода в JSON. `quests/manifest.json` перечисляет узлы сети и стартовый
узел; каждый узел описан отдельным файлом (`home-pc.json`,
`nortech-core.json`) со следующей структурой:

```jsonc
{
  "id": "home-pc",
  "displayName": "...",
  "requiredLevel": 0,
  "requiredKeyToConnect": "",
  "scanHints": ["..."],
  "directories": [
    { "path": "home", "hidden": false, "files": [
      { "name": "secret.txt", "content": "...", "encrypted": true,
        "cipherShift": 3, "grantsKeyOnDecrypt": "shadow42" }
    ]}
  ],
  "hack": {
    "targetName": "mainframe", "requiredKey": "shadow42",
    "code": "4216", "maxAttempts": 10, "pursuitSeconds": 90,
    "revealsDirectory": "system/secret", "grantsKey": "admin1234"
  },
  "connections": ["nortech-core"],
  "isEnding": false,
  "endings": []
}
```

`content` всегда хранится в открытом виде — если `encrypted: true`,
игра сама шифрует его сдвигом `cipherShift` при загрузке, так что
достаточно один раз написать текст на русском/английском без ручного
подсчёта шифра Цезаря.

Чтобы добавить новый узел сети: создать `quests/<id>.json` по этой
схеме и добавить имя файла в список `nodes` в `manifest.json`.

## Технологии

- C# / .NET 10
- [Spectre.Console](https://spectreconsole.net/) — цветной вывод в терминал
- [Figgle](https://github.com/drewnoakes/figgle) — ASCII-баннеры
- `System.Text.Json` — загрузка квестов и сохранений

## Статус проекта

Весь план (недели 1–4) реализован: сюжет, шифр Цезаря, сетевая карта
на JSON-квестах, мини-игра Mastermind с таймером слежки и четыре
концовки.
